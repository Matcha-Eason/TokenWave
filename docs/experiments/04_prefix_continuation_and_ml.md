# 前缀续写与机器学习增强实验

本实验补完 refined 四步方案里的第三步和第四步：

3. 前缀续写实验：只给 25%、50%、75% 前缀，用已有 NMF 波形基反推完整曲线，评估未来曲线和未来峰值。
4. 机器学习增强：把波形基权重、当前相位、近期斜率、事件密度喂给梯度提升模型，预测桌宠目标和科研目标。

## 输入

```text
outputs/coderforge_full/turn_timeseries.parquet
outputs/waveform_basis_experiment/nmf_basis.npy
outputs/waveform_basis_experiment/channel_scaling.csv
```

入口脚本：

```text
scripts/prefix_pet_target_experiment.py
```

正式实验命令：

```bash
MPLCONFIGDIR=.matplotlib \
PYTHONPATH=scripts \
LOKY_MAX_CPU_COUNT=8 \
.venv/bin/python scripts/prefix_pet_target_experiment.py \
  --input outputs/coderforge_full/turn_timeseries.parquet \
  --basis outputs/waveform_basis_experiment/nmf_basis.npy \
  --channel-scaling outputs/waveform_basis_experiment/channel_scaling.csv \
  --out-dir outputs/prefix_pet_target_experiment \
  --max-trajectories 30000
```

脚本优先使用 LightGBM；如果环境没有安装 `lightgbm`，自动降级到 sklearn 的 `HistGradientBoosting`。本机正式结果使用的是 sklearn fallback。

## 实验协议

每条轨迹固定抽三个前缀：

| 前缀 | 含义 |
|---|---|
| 25% | 只看早期四分之一 |
| 50% | 只看前半段 |
| 75% | 只看前三分之二到四分之三 |

波形主信号仍然只有三条：

| 中文名 | 字段 |
|---|---|
| 模型输出波 | `assistant_tokens` |
| 思考规划波 | `thinking_tokens_est` |
| 环境反馈波 | `user_tokens_est` |

事件只作为机器学习增强特征：

| 中文名 | 字段 |
|---|---|
| 编辑事件 | `edit_count` |
| 测试事件 | `test_count` |
| 错误反馈事件 | `error_feedback_count` |
| 结束事件 | `finish_call_count` |

## 第三步：前缀续写

前缀续写使用已有的 8 个 NMF 波形基。给定前缀后，脚本只用已观察部分求解 NMF 权重，再续写完整 64 点三通道曲线。

评估两个问题：

| 问题 | 指标 |
|---|---|
| 未来曲线像不像 | 未来段归一化均方误差 |
| 是否能预警未来新峰 | 未来峰值 F1 / balanced accuracy |

正式结果：

| 前缀 | 通道 | 平均模板未来误差 | NMF 前缀续写未来误差 | NMF 未来峰 F1 |
|---:|---|---:|---:|---:|
| 25% | 模型输出波 | 0.457 | 0.526 | 0.177 |
| 25% | 思考规划波 | 0.457 | 0.526 | 0.223 |
| 25% | 环境反馈波 | 0.457 | 0.526 | 0.001 |
| 50% | 模型输出波 | 0.423 | 0.501 | 0.272 |
| 50% | 思考规划波 | 0.423 | 0.501 | 0.024 |
| 50% | 环境反馈波 | 0.423 | 0.501 | 0.147 |
| 75% | 模型输出波 | 0.320 | 0.331 | 0.000 |
| 75% | 思考规划波 | 0.320 | 0.331 | 0.000 |
| 75% | 环境反馈波 | 0.320 | 0.331 | 0.003 |

### 第三步结论

纯 NMF 前缀续写没有在未来曲线误差上打过平均模板。这个结论比较明确：只靠“前缀拟合波形基再外推完整曲线”，还不能作为强预测器。

但 NMF 对“未来是否出现新峰”有一些信号，尤其是：

- 25% 前缀的思考规划峰，F1 约 0.223。
- 50% 前缀的模型输出峰，F1 约 0.272。
- 50% 前缀的环境反馈峰，F1 约 0.147。

这说明波形基更像“形态解释和弱预警特征”，而不是单独可用的未来曲线生成器。

## 第四步：机器学习增强

监督模型比较三组特征：

| 特征组 | 含义 |
|---|---|
| 统计特征 | 当前相位、近期斜率、累计量、事件密度 |
| 波形特征 | NMF 权重、前缀重构误差、NMF 续写出的未来波形摘要 |
| 增强特征 | 统计特征 + 波形特征 |

预测目标分两类：

| 目标 | 用途 |
|---|---|
| 未来 3 轮长输出 | 桌宠“认真工作/持续输出” |
| 未来 5 轮红温调试 | 桌宠“高压调试” |
| 5 轮内结束 | 桌宠收束参考，但固定前缀下极稀有 |
| 未来任一主通道新峰 | 波形研究目标 |
| 5 状态分类 | 桌宠状态识别 |
| 剩余轮数 | 科研目标，不直接展示给桌宠 |

正式结果摘要：

| 目标 | 统计特征 AUC/F1 | 波形特征 AUC/F1 | 增强特征 AUC/F1 |
|---|---:|---:|---:|
| 未来 3 轮长输出 | AUC 0.779 / F1 0.733 | AUC 0.753 / F1 0.732 | AUC 0.791 / F1 0.739 |
| 未来 5 轮红温调试 | AUC 0.682 / F1 0.732 | AUC 0.564 / F1 0.734 | AUC 0.681 / F1 0.731 |
| 未来任一新峰 | AUC 0.894 / F1 0.686 | AUC 0.861 / F1 0.647 | AUC 0.898 / F1 0.691 |
| 5 轮内结束 | AUC 0.778 / F1 0.667 | AUC 0.889 / F1 0.526 | AUC 0.999 / F1 0.667 |

剩余轮数科研目标：

| 特征组 | 平均绝对误差 | 中位绝对误差 | log R2 |
|---|---:|---:|---:|
| 统计特征 | 0.563 | 0.493 | 0.998 |
| 波形特征 | 0.478 | 0.463 | 0.998 |
| 增强特征 | 0.475 | 0.464 | 0.998 |

这里的剩余轮数结果不能和真实在线预测等同，因为固定前缀协议本身给了 25/50/75 这种相位信息，剩余长度更容易被推断。它可以作为科研对照，不适合直接宣传成桌宠可用能力。

5 状态分类在固定前缀实验中分数很高，但要谨慎解释：

| 特征组 | accuracy | macro F1 |
|---|---:|---:|
| 统计特征 | 0.9996 | 0.9990 |
| 波形特征 | 0.6935 | 0.2945 |
| 增强特征 | 0.9996 | 0.9990 |

固定 25/50/75 前缀下，测试集实际只出现 `deep_output`、`overheat_debugging`、`steady_work` 三类，几乎没有“读题理解”和“收束”。因此这个状态分类结果只能说明固定前缀点上的状态容易识别，不能替代前一份在线任意 turn 的桌宠状态实验。

## 特征解释

未来长输出最重要的特征：

- NMF 续写出的未来模型输出总量。
- 未来段重构误差。
- 累计编辑次数。
- 当前已观察轮数。
- NMF 第 8 个波形基权重。

未来红温调试最重要的特征：

- 错误反馈密度。
- 最近错误反馈均值。
- 最近测试均值。
- 当前模型输出长度。
- 累计错误反馈次数。

未来新峰最重要的特征：

- 累计思考长度。
- NMF 续写出的未来环境反馈总量。
- NMF 续写出的未来模型输出总量。
- 模型输出密度。
- 未来段重构误差。

剩余轮数科研目标最重要的特征：

- 前缀比例。
- 已观察轮数。
- NMF 续写出的未来模型输出总量。
- NMF 第 7 个波形基权重。
- NMF 第 2 个波形基权重。

## 当前结论

| 问题 | 结论 |
|---|---|
| 第三步是否完成 | 完成。纯 NMF 前缀续写已在 25/50/75 上验证。 |
| 第四步是否完成 | 完成。统计特征、波形特征、增强特征三组监督模型已对比。 |
| 纯波形基能否单独续写未来曲线 | 目前不行，未来曲线误差高于平均模板。 |
| 波形基是否完全没用 | 不是。它对未来峰值、长输出和剩余长度有弱到中等辅助价值。 |
| 最适合桌宠的一组目标 | 未来 3 轮长输出、未来 5 轮红温、在线 5 状态。 |
| 精确剩余轮数怎么处理 | 保留为科研目标，不作为桌宠直接展示。 |

## 对后续方案的影响

桌宠插件的第一版不应该依赖“纯数学波形续写”。

更合理的方案是：

```text
实时事件/逐轮统计
  -> 统计特征主导
  -> NMF/谐波特征辅助
  -> 输出长输出概率、红温概率、收束概率、桌宠状态
```

NMF 波形基适合放在解释层和辅助特征层：

- 解释当前任务像哪种历史波形。
- 判断未来是否可能出现新峰。
- 在机器学习增强模型里提供少量额外信号。

它暂时不适合作为独立的未来曲线预测器。
